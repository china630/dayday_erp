Модули и функции DayDay ERP

Сводка по PRD.md и TZ.md: целевой скоуп текущей линии продукта (ориентир v25.x) и запланированные горизонты v2 / v3 / v4+. Оценка % готовности — инженерно-продуктовая (реализовано относительно зафиксированного в документах объёма для данного горизонта), не аудит кода.

Условные обозначения

  • Горизонт «Текущая» — функциональность, входящая в актуальный скоуп ядра и расширений из PRD §4, §4.12–4.13 и синхронизированную с TZ (M1–M9, платформа).
  • v2 — PRD §5 (конкурентный scope: взаиморасчёты углублённо, налоговый экспорт, Multi-GAAP, ОС/ТМЦ).
  • v3 — PRD §6 (экосистема AZ: e-gov, ЭЦП, Direct Banking).
  • v4+ — PRD §7+ (коммерция, биллинг, квоты; часть уже внедрена как MVP платформы).
  • Критичность для системы: К1 — ядро (без этого продукт не выполняет роль ERP); К2 — высокая (основной операционный контур); К3 — средняя (конкурентность, удобство); К4 — низкая (расширения, интеграции).
